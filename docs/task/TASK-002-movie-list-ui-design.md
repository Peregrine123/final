# Task: 影单页面（电影列表页）UI/UX 美化设计

> 复制本文件为 `docs/task/TASK-002-movie-list-ui-design.md`，并在 `issue/README.md` 中登记。

## 任务内容
- **要做什么**：重新设计并美化电影列表展示页 (`MoviesDefault.vue`) 的 UI/UX。
- **为什么要做**：当前的电影列表页布局较旧（传统的左图右文或简单卡片），卡片样式单一，缺乏视觉吸引力，且交互（如悬停效果）不够现代化。
- **目标**：打造一个整洁、现代、视觉舒适的电影墙/列表页面，风格参考 Netflix 等现代流媒体平台，并采用瀑布流（masonry）布局提升浏览体验。
- 范围：仅限于 `blc-vue/src/components/library/MoviesDefault.vue` 组件及其样式调整。
- 约束：**严禁修改后端代码或数据库**。所有的 UI 改造必须基于现有的 `/movies` 接口返回的数据结构进行。

## 具体实施方案

### 1. 设计理念 (UI/UX Concept)
- **视觉焦点**：强调电影海报 (Poster Art)。
- **交互体验**：
    - **全卡片点击**：用户点击卡片任意位置均可跳转详情。
    - **无干扰浏览**：移除卡片上浮等大幅度动效，保持沉稳。
    - **无悬停解释**：鼠标移入卡片时不展示 Tooltip/Popover/遮罩等额外信息，避免干扰浏览。

### 2. 页面布局设计 (Layout Design)

#### A. 顶部搜索与筛选区
*   **搜索栏**：保持现有功能，视情况优化样式。

#### B. 电影瀑布流区 (Masonry / Waterfall)
*   **容器布局**：采用瀑布流（masonry）效果，卡片按列自上而下自然填充，整体观感接近现代流媒体平台的“海报墙”。
    *   优先：纯 CSS 实现（避免引入新依赖），例如 `column-count/column-gap` + `break-inside: avoid` 或等价方案。
    *   响应式：根据断点调整列数（例如移动端 2 列、平板 3-4 列、桌面 5-7 列，具体以实际效果为准）。
*   **电影卡片 (Movie Card)**：
    *   **点击区域**：整个卡片可点击。
    *   **封面**：满铺，圆角。
    *   **无悬停位移**：移除 `transform: translateY`。
    *   **无遮罩**：移除图片上的黑色遮罩层。
    *   **无悬停说明层**：不使用 `el-popover`/tooltip 等悬停解释层；仅保留点击进入详情页的交互。
    *   **允许的轻量反馈**：可保留非常轻微的非位移类反馈（例如边框/阴影/亮度微调），但不得出现“上浮/弹起”等位移动效。

#### C. 底部翻页区 (Pagination)
*   **分页组件**：居中展示，优化间距。

### 3. 色彩与排版 (Color & Typography)
- **背景**：保持清爽。
- **卡片文字/信息**：如需展示标题/年份等，建议直接在卡片下方/内部常驻展示（不依赖悬停）。

### 4. 技术实现细节
- **组件结构**：
    - 保持 `el-card` 结构简单，内部按需移除默认 padding。
- **图片加载**：`object-fit: cover`。

## 步骤
1.  **调整交互**：修改 `MoviesDefault.vue`，确保点击事件绑定在 Card 根元素。
2.  **移除特效**：删除 CSS 中的 `hover: translateY` 和 `.mask` 相关样式。
3.  **样式微调**：在不增加悬停解释层的前提下，微调卡片间距、圆角、阴影等观感（避免明显动效）。

## 过程验证
1.  **验证点：卡片样式**
    -   **观察**：卡片是否整齐排列，海报是否清晰，圆角是否生效。
    -   **通过标准**：视觉上整齐划一，无错位。
2.  **验证点：悬停交互**
    -   **观察**：鼠标移入卡片时，是否没有上浮/遮罩/Tooltip/Popover 等解释性反馈。
    -   **通过标准**：无干扰浏览（允许仅保留 `cursor: pointer` 等非侵入提示）。

## 最终结果验证
-   **验收项**：打开电影/图书列表页。
-   **通过标准**：页面呈现出类似 Netflix 等现代流媒体平台的海报墙 + 瀑布流观感；无卡片上浮位移、无悬停解释层。

## 风险与回滚
-   **风险点**：图片比例不一致导致卡片高度参差不齐。
-   **规避**：尽量统一海报展示宽高比（`object-fit: cover`），瀑布流下注意 `break-inside` 等避免卡片被切断。
