# TASK-015 博客内容页 UI/UX 重构

## 1. 背景与动机
当前的博客详情页 (`ArticleDetails.vue`) 使用的是基础的 "Markdown 查看器" 风格。虽然功能正常，但缺乏现代数字出版物（如 Medium, 掘金, 知乎专栏）的沉浸感、可读性和精致度。本次重构的目标是将其转化为专业、沉浸式的阅读体验。

## 2. 设计目标
*   **现代美学**：从 "渲染的文本文件" 转变为 "杂志级文章" 外观。
*   **增强可读性**：优化排版（行长、字体搭配、间距），以适应长文阅读。
*   **增强互动**：添加目录 (TOC)、社交分享以及清晰的行动号召 (点赞/收藏)。
*   **响应式设计**：确保在移动端、平板和桌面端都有完美体验。

## 3. 详细 UI/UX 规范

### 3.1. 布局结构
采用 **响应式网格布局**：
*   **桌面端 (>= 1024px)**：双栏布局。
    *   **主内容区 (左/中)**：宽度约 700px - 800px。专注于阅读区域。
    *   **侧边栏 (右)**：宽度约 300px。吸顶定位 (Sticky)。包含目录和作者互动信息。
*   **移动/平板 (< 1024px)**：单栏布局。
    *   目录移至可折叠的抽屉或顶部 "快速跳转" 菜单。
    *   侧边栏元素移至底部（内容下方）。

### 3.2. 组件拆解

#### A. 沉浸式 Hero 头部区
替代简单的横幅，创建一个丰富的信息头：
*   **背景**：高质量封面图（带模糊/遮罩以保证文字对比度），若无封面则使用简洁的极简渐变。
*   **元素**：
    *   **标题**：大号、加粗的衬线或展示型无衬线字体 (例如 2.5rem+)。
    *   **元数据**：
        *   **作者**：头像 (圆形) + 姓名 + "关注" 按钮 (小)。
        *   **日期**：格式化日期 (例如 "2026年1月17日")。
        *   **阅读时间**：预估时间 (例如 "预计阅读 5 分钟")。
        *   **浏览/点赞**：图标 + 计数。

#### B. 阅读体验 (正文区域)
*   **排版**：
    *   **正文**：使用高可读性的衬线字体 (如 *Noto Serif SC*, *Songti SC*) 或现代几何无衬线字体 (如 *Inter*, *PingFang SC*)，行高宽松 (1.7 - 1.9)。
    *   **标题**：加粗，层级分明。鼠标悬停时显示锚点链接。
*   **视觉元素**：
    *   图片宽度应大于文本列（打破网格）或在适当位置全宽显示。
    *   图片下方配有说明文字样式。
*   **代码块**：
    *   Mac 风格窗口头（红/黄/绿圆点）。
    *   右上角 "复制" 按钮。
    *   暗色主题语法高亮。
*   **引用**：
    *   风格化的块引用，带有粗边框强调或巨大的引号图标。

#### C. 侧边栏 (仅桌面端)
*   **吸顶目录 (TOC)**：
    *   根据文章中的 H1-H3 标签自动生成。
    *   随滚动自动高亮当前阅读章节 (ScrollSpy)。
*   **作者卡片**：
    *   简介。
    *   社交链接。
*   **互动操作**：
    *   悬浮工具栏 (或位于侧边栏)：
        *   点赞 (爱心) 按钮，带动画。
        *   书签/收藏 按钮。
        *   分享 (生成链接/海报)。

#### D. 底部区域 (文章结束后)
*   **标签**：胶囊样式的标签。
*   **互动**："这篇文章有帮助吗？" 或大号点赞按钮。
*   **导航**："上一篇" 和 "下一篇" 卡片，带缩略图。
*   **评论区**：
    *   现代列表设计。
    *   富文本评论编辑器。

## 4. 技术实施计划

### 阶段 1: 结构与布局重构
1.  **提取组件**：
    *   创建 `BlogHero.vue` (头部信息)。
    *   创建 `BlogContent.vue` (主要文本渲染器)。
    *   创建 `BlogSidebar.vue` (侧边栏容器)。
    *   创建 `BlogTOC.vue` (目录逻辑)。
2.  **网格系统**：在 `ArticleDetails.vue` 中实现 CSS Grid 或 Flexbox 容器以支持双栏布局。

### 阶段 2: 内容解析与增强
由于内容是 HTML (`v-html`) 格式，需要客户端处理：
1.  **目录生成**：组件挂载后，在内容 ref 中查询 `querySelectorAll('h1, h2, h3')` 以构建 TOC 数据数组。
2.  **注入锚点**：动态为标题添加 `id` 属性以便链接跳转。
3.  **代码块增强**：使用指令或 `onMounted` 逻辑将 "复制" 按钮追加到 `<pre>` 块中。

### 阶段 3: 样式 (打造 "现代感")
1.  **排版系统**：定义博客专用的字体和间距 CSS 变量。
2.  **主题支持**：确保颜色在亮色模式（及潜在的暗色模式）下表现良好，优先保证亮色模式。
3.  **动画**：
    *   目录平滑滚动。
    *   图片渐入效果。
    *   按钮的微交互。

## 5. 验收标准
*   [ ] 页面布局能正确适配移动端与桌面端。
*   [ ] 目录准确，可点击，并能高亮显示当前章节。
*   [ ] 文章文本清晰易读，排版与 UI 界面区分明显。
*   [ ] "点赞" 和 "收藏" 按钮易于访问且反馈明显。
*   [ ] 相邻文章导航功能正常。
*   [ ] 现有的 Markdown 内容（表格、图片、代码）样式无崩坏。
