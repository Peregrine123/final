# TASK-006 博客页面现代化UI重构

> 约束：本文档只写“技术与方案描述 / 验证方式 / 结果”，不要粘贴任何代码片段（含配置片段/命令行脚本/SQL）。

## 任务内容
- **要做什么**：重构博客列表页 (`Articles.vue`) 和博客详情页 (`ArticleDetails.vue`) 的 UI 设计。
- **为什么要做（背景/问题）**：当前设计风格较为陈旧（例如过度使用的倾斜切角、复杂的 hover 3D 变换），与现代 Web 设计强调的“简洁、内容优先、扁平化/微质感”趋势不符。
- **目标（可量化）**：
    1.  实现一个现代化的卡片式列表布局（Grid 响应式）。
    2.  提升文章详情页的阅读体验（排版优化、Hero Banner 头部）。
    3.  统一视觉风格，减少过时的 CSS 特效，增加留白和呼吸感。
- **范围（包含/不包含）**：
    -   包含：`Articles.vue`, `ArticleDetails.vue` 的 Template 和 Style 修改。
    -   不包含：后端 API 修改，数据库结构修改。
- **影响面（模块/接口/数据/用户）**：
    -   影响用户端博客浏览体验。
    -   不影响数据接口。

## 具体实施方案

本次重构将采用“内容优先”的设计理念，移除干扰视觉的装饰性元素，使用 CSS Grid 配合自定义 CSS 实现响应式布局。

### 1. 博客列表页 (`Articles.vue`) 重构方案

*   **布局调整**：
    *   放弃原有的单列复杂卡片堆叠。
    *   采用 **响应式网格布局**：
        *   PC 端：3列或4列 Grid 布局 (Masonry 风格或等高卡片)。
        *   平板：2列。
        *   移动端：1列。
*   **卡片设计**：
    *   **风格**：极简风格，圆角 (Border-radius 8px-12px)，轻微阴影 (Box-shadow)，背景纯白。
    *   **封面图**：保持 16:9 或 4:3 比例，取消原有的倾斜遮罩，图片使用 `object-fit: cover` 居中裁剪。
    *   **排版**：
        *   标题：加粗，字号适中，限制行数（超出省略）。
        *   摘要：颜色较淡，行高 1.6，限制 3 行。
        *   元信息（日期、作者）：移至底部或标题下方，使用小字号灰色文本。
    *   **交互**：
        *   Hover 时整体卡片轻微上浮 (TranslateY -5px) 并加深阴影，取消原有的图片旋转放大效果。
*   **分页组件**：保持 Element UI 分页组件，调整间距使其更具呼吸感。

### 2. 博客详情页 (`ArticleDetails.vue`) 重构方案

*   **头部设计 (Header)**：
    *   引入 **Hero Image** 概念：如果文章有封面图，将其作为顶部宽幅背景或大图展示，增加视觉冲击力。
    *   标题区域：居中或左对齐的大号标题，下方跟随作者头像、发布日期、阅读量等元信息。
*   **内容区域 (Content)**：
    *   **容器**：限制最大阅读宽度 (Max-width: 800px)，保证在大屏下的阅读舒适度（避免视线移动距离过长）。
    *   **排版**：
        *   正文：字号 16px-18px，行高 1.8，深灰色 (#333) 而非纯黑。
        *   段落间距：增加段落间距 (Margin-bottom: 1.5em)。
    *   **Markdown 样式**：优化 `markdown-body` 的 CSS，确保引用、代码块、列表的样式与整体风格统一（简洁、清爽）。
*   **底部导航**：
    *   增加“返回列表”按钮（悬浮或底部固定位置）。

### 步骤
1.  **备份与清理**：备份现有 CSS，清理 `Articles.vue` 中过时的 CSS 样式（如 skew, rotate 变换）。
2.  **列表页重构**：
    *   引入 **CSS Grid** 实现响应式网格布局，避免传统浮动/栅格在“卡片高度不一致”场景出现大面积空白。
    *   重写 `.blog-card` 样式，实现现代化卡片风格。
    *   调整图片容器，确保响应式适配。
3.  **详情页重构**：
    *   重构头部结构，实现 Hero Banner 布局。
    *   调整内容容器宽度和 Padding。
    *   优化文字排版 CSS。
4.  **适配测试**：在不同分辨率（桌面、手机）下检查布局崩坏情况。

## 过程验证

1.  **验证点：列表页响应式**
    *   **观察/指标**：改变浏览器宽度。
    *   **通过标准**：
        *   > 1200px 时显示 3-4 列。
        *   768px - 1200px 显示 2 列。
        *   < 768px 显示 1 列且左右留白合适。
        *   卡片高度对齐或布局整齐，无错位。
2.  **验证点：详情页阅读体验**
    *   **观察/指标**：打开一篇长文。
    *   **通过标准**：正文文字清晰，行距舒适，图片不溢出容器，在手机端无需横向滚动。

## 最终结果验证
- **验收项**：博客列表页、博客详情页 UI。
- **通过标准**：视觉风格统一，无明显 CSS 渲染错误，交互流畅（Hover 效果不卡顿）。
- **回归范围**：博客模块的所有页面跳转、分页功能。

## 结果（已落地内容）
- 已完成 UI 现代化重构，涉及文件：
  - `blc-vue/src/components/Blog/Articles.vue`
  - `blc-vue/src/components/Blog/ArticleDetails.vue`
- 为了便于演示新样式，补充了影评种子文章脚本：`blc/seed-blog-2026.sql`（可选导入）。

## 风险与回滚
- **风险点**：Markdown 内容中的图片或特殊 HTML 标签可能破坏新的布局（如超宽图片）。
- **规避手段**：在 CSS 中强制设置详情页内容图片 `max-width: 100%`。
- **回滚策略**：使用 Git 恢复修改前的文件版本。
