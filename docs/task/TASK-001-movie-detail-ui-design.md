# Task: 影评页面（电影详情页）UI/UX 美化设计

> 复制本文件为 `docs/task/TASK-001-movie-detail-ui-design.md`，并在 `issue/README.md` 中登记。

## 任务内容
- **要做什么**：重新设计并美化电影详情页 (`MovieDetails.vue`) 的 UI/UX。
- **为什么要做**：当前的电影详情页布局过于简单，信息展示层级不清晰，缺乏视觉吸引力，无法提供沉浸式的浏览体验。
- **目标**：打造一个现代化、布局合理、视觉美观且信息丰富度的电影详情页面，提升用户体验。
- 范围：仅限于 `blc-vue/src/components/library/MovieDetails.vue` 组件及其样式调整。
- 约束：**严禁修改后端代码或数据库**。所有的 UI 改造必须基于现有的 `/movies` 接口返回的数据结构进行。如果现有数据不足以支撑设计（如缺少详细演员表、评分等），前端应使用默认值、静态 Mock 数据或隐藏相关模块，**绝对不能**因此要求修改后端接口。
- 影响面：前端电影详情展示模块。

## 具体实施方案

本次设计旨在通过现代化的布局和视觉元素，提升页面的质感。

### 1. 设计理念 (UI/UX Concept)
- **沉浸感 (Immersion)**：利用电影海报生成模糊背景（Backdrop），营造氛围感。
- **层级清晰 (Hierarchy)**：通过字体大小、颜色深浅和卡片式布局，区分标题、元数据、简介和互动区域。
- **交互性 (Interactivity)**：增加明显的行动按钮（如“写影评”、“收藏”），并为可点击元素增加 Hover 效果。

### 2. 页面布局设计 (Layout Design)

页面将分为三个主要部分：

#### A. 顶部沉浸式头部 (Hero Section)
*   **背景**：使用当前电影封面图的高斯模糊 (Gaussian Blur) 版本铺满顶部区域，叠加半透明深色遮罩，保证文字可读性。
*   **内容容器**：垂直居中展示核心信息。
    *   **左侧**：高清电影海报 (Poster)，带阴影 (Box Shadow) 和圆角。
    *   **右侧**：
        *   **标题**：超大字号 (H1)，白色。
        *   **原名/外文名**：较小字号，灰色。
        *   **基础元数据**：年份 | 类型 | 时长 (使用分隔符布局)。
        *   **评分展示**：使用星星组件 (Rate) 展示评分 (如有数据) 或预留位置。
        *   **操作栏**：一组按钮，“收藏”、“写影评”、“分享”。

#### B. 主体内容区 (Main Content) - 左侧 70%
*   **剧情简介 (Synopsis)**：
    *   标题：“剧情简介”。
    *   内容：增加行高 (line-height) 和段间距，提升阅读体验。
*   **演职员表 (Cast & Crew)**：
    *   使用横向滚动或网格布局展示导演和主要演员。
    *   头像 + 名字 + 饰 角色名。
*   **短评/影评 (Reviews)** (UI预留)：
    *   列表形式展示用户评论。
    *   包含用户头像、昵称、评分星级、评论内容、时间。

#### C. 侧边信息栏 (Sidebar) - 右侧 30%
*   **详细信息 (Details)**：
    *   导演 (Director)
    *   编剧 (Writer)
    *   主演 (Starring) - 文本列表
    *   类型 (Genre)
    *   制片国家/地区
    *   语言
    *   上映日期
*   **相关推荐 (Related)** (UI预留)：
    *   展示几部同类型电影的小海报。

### 3. 色彩与排版 (Color & Typography)
- **主色调**：以白色 (#FFFFFF) 为背景，深灰 (#333333) 为主文字色。
- **强调色**：使用 Element UI 的 Primary 色 (#409EFF) 或暖色调 (如 #FF9900) 用于评分和按钮。
- **字体**：优先使用系统无衬线字体 (PingFang SC, Roboto, Helvetica Neue, sans-serif)。

### 4. 技术实现细节
- **数据适配 (CRITICAL)**：
    - 现有接口 `/movies` 返回字段有限（通常只有 `title`, `cover`, `cast`, `date`, `press`, `summary`）。
    - 对于设计中存在但接口未返回的字段（如 `rating`, `actors_images`, `reviews`）：
        - **方案 A**：使用写死的静态数据或随机 Mock 数据（仅用于展示 UI 效果）。
        - **方案 B**：使用 `v-if` 隐藏这些模块。
        - **方案 C**：对 `cast` 字段（导演/主演）进行简单的字符串解析（如果它是逗号分隔的字符串），尝试拆分展示，但不依赖后端结构化。
- 使用 Element UI 的 `el-row`, `el-col` 实现响应式栅格布局。
- 使用 CSS `filter: blur()` 实现背景模糊。
- 使用 `z-index` 管理层级，确保内容在背景之上。


## 步骤
1.  **重构 HTML 结构**：将 `MovieDetails.vue` 原有的简单 `div` 结构改为基于 `el-container` 或 `el-row/el-col` 的分层结构。
2.  **实现 Hero 区域**：
    *   添加背景容器，设置动态 style 绑定背景图。
    *   布局海报和右侧核心信息。
    *   添加“收藏”等按钮组件。
3.  **实现主体内容区**：
    *   美化“简介”部分文本样式。
    *   创建“演职员”和“评论”的静态 UI 占位组件。
4.  **实现侧边栏**：
    *   使用 `el-card` 或列表样式展示详细元数据。
5.  **样式微调**：调整间距 (Padding/Margin)、圆角 (Border Radius) 和阴影 (Box Shadow)。

## 过程验证
1.  **验证点：布局结构**
    -   **观察**：页面是否明显分为顶部 Hero、左侧内容、右侧边栏。
    -   **通过标准**：各区域边界清晰，无错位。
2.  **验证点：视觉效果**
    -   **观察**：顶部背景是否模糊且美观，海报是否有阴影，文字是否清晰可读。
    -   **通过标准**：符合“沉浸感”设计要求。
3.  **验证点：响应式**
    -   **观察**：改变浏览器宽度，布局是否会自动调整（如侧边栏掉落或重排，虽然主要针对 PC，但应保持基本不崩坏）。
    -   **通过标准**：内容不溢出，无横向滚动条。

## 最终结果验证
-   **验收项**：打开任意一部电影的详情页。
-   **通过标准**：页面展示出一张美观的电影详情卡片，包含海报、背景、清晰的信息排版，且控制台无报错。
-   **回归范围**：确保从列表页点击进入详情页的功能正常。

## 风险与回滚
-   **风险点**：CSS 样式冲突，导致布局在不同分辨率下崩坏。
-   **风险点**：现有 API 数据字段过少，导致新 UI 大部分区域为空白或只能显示 Mock 数据。
-   **规避手段**：使用 Scoped CSS；在数据缺失时提供美观的默认占位图或文案（Empty State）。
-   **回滚策略**：Git Revert 相关的 CSS 和 Vue 文件修改。
